## 一、业务活动与状态机（当前实现）

### 1.1 活动节点与页面对照

| 序号 | 活动节点 | 角色 | Web 路由 | 主要 API | 说明 |
|------|----------|------|----------|----------|------|
| 1 | 需求信息确认 | 企业 | `/enterprise/demand/preview` | `POST /api/projects/demand/create` | 创建草稿、保存草稿、上传材料 |
| 2 | 中试需求提交 | 企业 | `/enterprise/demand/submit` | `POST .../demand/submit` | 须填需求内容、11 位联系电话 |
| 3 | 需求受理登记 | 调度员 | `/center/dispatch/demand/workbench` | `POST .../accept-register` | 增强版工作台：核对概要后登记 |
| 3a | 需求退回通知 | 调度员 | `/center/dispatch/demand/reject` | `POST .../reject` | **仅 SUBMITTED** 可退回 |
| 3b | 需求材料补充 | 企业 | `/enterprise/demand/progress` 内嵌 | `POST .../supplement` | 进度页内补充；独立页 `/supplement` 仍保留 |
| 4 | 受理材料审核 | 审核员 | `/center/audit/demand/verify` | `POST .../verify` | **核验与受理决定一步完成** |
| 5 | 受理回执签收 | 企业 | `/enterprise/demand/progress` 内嵌 | `POST .../receipt` | 进度页内签收；独立页 `/receipt` 仍保留 |
| 6 | 受理信息归档 | 调度员 | 工作台内「确认归档」 | `POST .../demand/archive` | 企业签收后 stage→EVALUATION |
| 7 | 查看受理进度 | 企业 | `/enterprise/demand/progress` | `GET .../demand/progress` | 含流程日志、五模块进度 |
| — | 我的需求项目 | 企业 | `/enterprise/demand/projects` | `GET /api/demand/enterprise/projects` | 阶段筛选 + 关键词搜索 |

> **与旧版差异：** 主路径不再经过 `VERIFYING` 子状态及独立「受理结果录入页」；审核员在「受理材料审核页」一次完成材料核验与同意/不予受理决定。

### 1.2 子状态流转（主路径）

```
DRAFT → SUBMITTED → ACCEPTING → ACCEPTED → RECEIPTED → ARCHIVED
                                                      ↓
                                              stage=EVALUATION
```

### 1.3 否支 ↺ 流转

| 触发 | 操作方 | 状态变迁 | 后续 |
|------|--------|----------|------|
| 调度员退回 | dispatcher | SUBMITTED → RETURNED | 企业补充 → SUBMITTED ↺ 工作台 |
| 审核员材料不齐 | auditor | ACCEPTING → RETURNED | 同上 |
| 审核员不同意受理 | auditor | ACCEPTING → CLOSED | 流程终止 |
| 企业补充材料 | enterprise | RETURNED → SUBMITTED | ↺ 调度员受理登记 |

### 1.4 各角色待办过滤（`DemandStatus.*Todos()`）

| 角色 | 待办 status | 典型操作 |
|------|-------------|----------|
| 企业 | DRAFT, RETURNED, ACCEPTED | 预览/补充/签收 |
| 调度员 | SUBMITTED, ACCEPTING, RECEIPTED | 受理登记/归档 |
| 审核员 | ACCEPTING | 材料审核 |

## 三、业务流程测试用例（场景级）

### DM-FLOW-001 主路径 — 同意受理至需求段归档

| 属性 | 内容 |
|------|------|
| 优先级 | P0 |
| 类型 | 主路径 |
| 前置 | 新建项目或清空 H2 |
| 自动化 | `npm run demand:flow`（至 RECEIPTED）；归档步骤需手工或扩展脚本 |

| 步骤 | 角色 | 操作 | 预期结果 |
|------|------|------|----------|
| 1 | enterprise | 需求预览页：填写项目名称、类型、周期、联系人、**11 位手机**、需求说明；上传 ≥1 份材料；保存草稿 | status=DRAFT，材料已关联 |
| 2 | enterprise | 进入需求填报页 → 提交需求 | status=**SUBMITTED**；调度员待办 +1 |
| 3 | dispatcher | 需求受理工作台：待办列表选中项目；核对申请概要、企业信息、材料清单 | 详情只读展示完整 |
| 4 | dispatcher | 填写受理备注（可选）→ 确认受理登记 | status=**ACCEPTING**；审核员待办 +1 |
| 5 | auditor | 受理材料审核页：下载核对附件 → 填写意见 →「材料齐全，同意受理」 | status=**ACCEPTED**；企业待办「回执签收」 |
| 6 | enterprise | 受理进度详情页 → 确认签收 | status=**RECEIPTED**；调度员待办「待归档」 |
| 7 | dispatcher | 工作台打开该项目 → 确认归档 | status=**ARCHIVED**，stage=**EVALUATION**，currentNode=评估前置核查 |
| 8 | enterprise | 我的需求项目 → 查看进度 | 需求段日志完整；主模块进入「评估中」 |
