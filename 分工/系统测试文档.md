## 一、文档说明

### 1.1 目的

本文档为毕业设计 **S8 系统测试** 的完整交付物，用于：

1. 指导测试人员按模块、按角色、按流程执行功能测试；
2. 记录缺陷与修复状态，确保三端（Web / 小程序 / 后端）联调可用；
3. 验证业务状态机与活动图逻辑符合正常系统操作规范；
4. 为论文第 6 章「系统测试」提供用例与结果素材。

### 1.2 测试范围

| 层次 | 范围 | 说明 |
|------|------|------|
| 后端 | `server/` Spring Boot API、状态机、JWT 鉴权 | 含 5 个业务模块 + 公共接口 |
| Web | `web/` 57 页 Vue 3 页面 | 四角色门户 + 公共页 |
| 小程序 | `miniapp/` 22 页 uni-app | 企业 16 页 + 技术人员 6 页 |
| 数据库 | H2 文件库（开发默认） | `server/data/zhongshi.mv.db` |
| 不在范围 | 微信真机 OAuth、WebSocket 实时推送、MySQL 生产部署 | 见总体规划 §10.1 简化策略 |

### 1.3 测试通过标准

- **P0 主链**：单项目从需求创建到 `stage=CLOSED` 可走通（见 TC-X1）；
- **否支 ↺**：五条活动图各至少 1 条退回/重派/补充路径可闭环；
- **单元测试**：`server/src/test` 下 5 个 ServiceTest 全部通过；
- **页面覆盖**：79 页均可访问，表单可提交或只读展示符合角色；
- **双端同步**：小程序关键操作后 Web 待办同步更新；
- **无阻塞缺陷**：无 P0/P1 级未修复缺陷（见 §八 缺陷清单）。

## 二、测试环境

### 2.1 软硬件环境

| 项 | 要求 |
|---|---|
| OS | Windows 10/11 或 macOS |
| JDK | 17+ |
| Node.js | 18+ |
| Maven | 3.8+ |
| 浏览器 | Chrome / Edge 最新版 |
| 微信开发者工具 | 最新稳定版（小程序测试） |
| HBuilderX | 可选，用于 uni-app 编译 |

### 2.2 启动步骤

```bash
# 终端 1 — 后端（默认 H2，端口 8080）
cd server
mvn spring-boot:run

# 终端 2 — Web 前端（端口 5173，代理 /api → 8080）
cd web
npm install
npm run dev

# 终端 3 — 小程序（可选）
cd miniapp
npm install
npm run dev:mp-weixin
# 微信开发者工具导入 miniapp/unpackage/dist/dev/mp-weixin
# 勾选：详情 → 本地设置 → 不校验合法域名
```

## 八、缺陷清单（代码审查 + 逻辑分析）

> 执行测试时按此清单逐项验证；修复后在「状态」列更新。

| 编号 | 严重度 | 模块 | 描述 | 影响 | 建议修复 | 状态 |
|------|--------|------|------|------|----------|------|
| BUG-01 | **P0** | 调度 | `DispatchStatus.dispatcherTodos()` 未包含 `PENDING_RECEIVE`，派单通知后调度员首页丢失待办 | 调度员需手动找项目继续跟进 | 在 `dispatcherTodos()` 中加入 `PENDING_RECEIVE` | 待修复 |
| BUG-02 | **P1** | 反馈 | `FeedbackController.review()` 硬编码 `request.setPassed(true)` | 复核确认无法拒绝 | 读取请求体中的 passed 字段 | 待修复 |
| BUG-03 | **P1** | 档案 | `ArchiveStatus.auditorTodos()` 含 `COLLECTED`，归集后审核员出现错误待办 | 待办指向台账页而非统计 | 从 auditorTodos 移除 COLLECTED | 待修复 |
| BUG-04 | **P1** | 调度 | `enterpriseTodos()` 未含 `PROGRESS_ACKED`，企业确认进度后丢失待办 | 企业无法继续查看进度 | 加入 PROGRESS_ACKED | 待修复 |
| BUG-05 | **P1** | 通用 | 步骤条用 `enum.ordinal()` 计算，退回状态显示错误 | RETURNED/AUDIT_REJECTED 等步骤条错位 | 改用显式 step 映射 | 待修复 |
| BUG-06 | **P2** | 调度 | `execConfirm` 允许从 CONFIRMED 直接确认，跳过进度填报 | 流程可 shortcut | 增加前置状态校验 | 待修复 |
| BUG-07 | **P2** | 反馈 | `passed=null` 时反馈模块视为通过，评估模块视为不通过 | 语义不一致 | 统一 null 处理策略 | 待修复 |
| BUG-08 | **P2** | 安全 | `/uploads/**` 无需认证即可访问 | 附件 URL 泄露后可被下载 | 改为鉴权代理或签名 URL | 待修复 |
| BUG-09 | **P2** | 前端 | 后端返回 HTTP 200 + code:401，axios 仅处理 HTTP 401 | Token 失效不自动跳转登录 | 拦截器增加 body.code 判断 | 待修复 |
| BUG-10 | **P2** | 小程序 | 无 `/api/files/upload` 封装，需求/评估页无法上传材料 | 小程序只能复用已有 materials | 增加 uni.uploadFile 封装 | 待修复 |
| BUG-11 | **P2** | 需求 | `nextProjectNo()` 非年度隔离、非并发安全 | 高并发可能编号重复 | 加锁或 DB 序列 | 待修复 |
| BUG-12 | **P3** | 评估 | `conclusion()` 未写 workflow_log | 进度时间线缺节点 | 补充 transition 调用 | 待修复 |
| BUG-13 | **P3** | 调度 | `NOTICED` 枚举未使用，实际用 `PENDING_RECEIVE` | 代码可读性 | 清理或统一 | 待修复 |
| BUG-14 | **P3** | 安全 | Controller 层无 `@PreAuthorize`，仅靠 Service 层校验 | 防御深度不足 | 毕设可接受，生产需加固 | 已知 |
| BUG-15 | **P3** | 安全 | 跨企业 IDOR：中心角色可读任意 projectId | 数据隔离不严 | 增加企业归属校验 | 已知 |

**项目主阶段 stage：** DEMAND → EVALUATION → DISPATCH → FEEDBACK → ARCHIVE → CLOSED

**需求 sub_status：** DRAFT → SUBMITTED → ACCEPTING → VERIFYING → (RETURNED ↺) → ACCEPTED → RECEIPTED → ARCHIVED

**关键 ↺ 路径：**
- 需求：RETURNED → SUBMITTED（材料补充）

- 反馈：AUDIT_REJECTED → 修改；FEEDBACK_AUDIT_REJECTED → 意见反馈
