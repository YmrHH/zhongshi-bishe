# 原型图说明

## 1. 生成范围

> 说明：本版主要依据《原型图生成规范.md》§6 全量页面清单与文字版业务节点生成，未逐张人工复核业务活动图 PNG，请在提交前按活动图 PNG 再做一次人工核对。


- 共生成 79 个页面文件，其中 Web 57 页，小程序 22 页。
- 每个页面均包含同名 `.html` 源文件与 `.png` 原型图。
- Web 页面采用 Element Plus 风格的蓝白后台布局；小程序页面采用微信绿色/白色简洁布局。

## 2. 分辨率与命名

- Web PNG：1920×1080。
- 小程序 PNG：750×1624，按 375×812 的 2 倍清晰度导出，便于论文插图和答辩展示。
- 文件名遵循 `[端别]-[角色简称]-[模块简称]-[页面简称].png`。

## 3. 关键跳转与闭环

- 图3-1 中试需求管理：材料不齐→退回意见页→材料补充页↺需求填报页；同意受理→受理结果录入→回执签收→归档→进度详情
- 图3-2 中试评估管理：不具备条件↺整改通知→条件材料补充↺前置核查；可行性否↺评估材料补充↺可行性审查
- 图3-3 中试调度管理：匹配失败↺资源匹配；异常需重派↺任务接收页；企业仅查看进度，不填报
- 图3-4 中试反馈管理：报告审核/复核否↺结果修改↺结果提交；反馈审核否↺复核意见反馈
- 图3-5 中试档案管理：台账不完整↺台账维护；统计数据不可用↺周期统计

## 5. 使用建议

- 写论文第 4 章时，建议从每个模块各选 1—2 张典型 PNG 插入，如需求填报、需求受理工作台、条件评估、任务派发、结果提交、报告审核、周期统计、简报查看等。
- 若后续要做 Axure/Figma，可直接按 HTML 结构或 PNG 版式复刻。
