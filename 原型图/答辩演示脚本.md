# 答辩演示脚本（S8）

> 建议时长 **12—15 分钟**：系统概述 2 分钟 + 主链演示 8 分钟 + 小程序/统计 3 分钟 + 总结 2 分钟。  
> 演示前删除 H2 数据文件 `server/data/zhongshi*` 或使用全新库，保证从空库开始可走完整流程。

## 一、启动与环境

1. 终端 1：`cd server && mvn spring-boot:run` → 打开 http://localhost:8080/doc.html
2. 终端 2：`cd web && npm run dev` → http://localhost:5173
3. （可选）HBuilderX 打开 `miniapp/`，微信开发者工具，勾选「不校验合法域名」

## 二、开场（约 2 分钟）

**讲解要点**：

- 题目：广州生产力促进中心 **中试服务管理系统**
- 三端：Web 中心管理端（调度员/审核员）+ 企业/技术人员门户 + uni-app 小程序
- 五模块状态机：需求 → 评估 → 调度 → 反馈 → 档案 → 结案
- 技术栈：Spring Boot 3 + Vue 3 + Element Plus + uni-app + JWT + H2

**可展示**：登录页四角色分流；中心管理工作台待办概览。

## 三、主链演示（约 8 分钟）

按 **P0 可演示主链** 操作，每步切换浏览器窗口/无痕标签对应角色。

### 3.1 需求模块（enterprise → dispatcher → auditor → enterprise）

| 顺序 | 账号 | 页面/操作 | 话术要点 |
|---:|---|---|---|
| 1 | enterprise | 需求预览 → 填报 → 提交 | 企业在线提交中试需求 |
| 2 | dispatcher | 需求受理工作台 → 受理 | 调度员登记受理 |
| 3 | auditor | 材料核验 → 受理结果录入 | 审核员核验材料并通知结果 |
| 4 | enterprise | 受理回执签收 | 企业确认回执，需求模块闭环 |

**否支演示（可选 30 秒）**：dispatcher 退回 → enterprise 材料补充 ↺ 重新提交。

### 3.2 评估模块（dispatcher → auditor → enterprise）

| 顺序 | 账号 | 操作 |
|---:|---|---|
| 5 | dispatcher | 评估前置核查 → 条件具备 |
| 6 | dispatcher | 可行性审查 → 可行 |
| 7 | auditor | 评估结论 → 通过 |
| 8 | enterprise | 结论签收 |

### 3.3 调度模块（dispatcher → technician → dispatcher → enterprise 只读）

| 顺序 | 账号 | 操作 |
|---:|---|---|
| 9 | dispatcher | 资源匹配 → 任务派发（选 technician） |
| 10 | technician | 任务接收 → 确认签收 → 进度填报 100% |
| 11 | dispatcher | 执行结果确认 → 调度归档 |
| 12 | enterprise | 调度进度查看（强调企业仅查看） |

### 3.4 反馈模块（technician → auditor → enterprise → auditor）

| 顺序 | 账号 | 操作 |
|---:|---|---|
| 13 | technician | 结果提交 |
| 14 | auditor | 报告审核 → 复核确认 |
| 15 | enterprise | 复核意见反馈 |
| 16 | auditor | 反馈审核 → 案例归档 |

### 3.5 档案模块（dispatcher → auditor → enterprise）

| 顺序 | 账号 | 操作 |
|---:|---|---|
| 17 | dispatcher | 台账维护 → 周期统计 / 成功率分析（ECharts） |
| 18 | auditor | 简报审核 |
| 19 | enterprise | 简报查看 |
| 20 | auditor | 档案归档 → 项目 **CLOSED** |

**收尾检查**：enterprise 首页步骤条五模块均已完成；消息中心有流程通知。

## 四、小程序演示（约 3 分钟）

1. **enterprise** 登录小程序 → 首页待办与 Web 一致
2. 点击待办跳转业务页（如材料补充、结论签收）
3. **technician** 登录 → 任务接收 / 进度填报（与 Web 同步）
4. 说明：调度员/审核员仅 Web 端，符合原型 B1 约束

## 七、常见问题预案

| 问题 | 应对 |
|---|---|
| 待办不刷新 | F5 或等 30s 轮询；检查后端是否运行 |
| 小程序登录失败 | 确认 8080 端口、勾选不校验合法域名 |
| 技术人员归档后仍有进度待办 | 已修复：`DispatchService` 按 stage 过滤 |
| 图表无数据 | 至少完成 1 个项目至 ARCHIVE 后统计页有数据 |
